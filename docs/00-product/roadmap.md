# Product roadmap

## Phase 0 — Foundation

- Confirm Figma screen inventory.
- Create semantic tokens and component library.
- Establish monorepo, CI, linting, contracts, and demo provider.
- Implement business profile and core schemas.

## Phase 1 — Content assistant MVP

- Authentication.
- Onboarding.
- Chat and structured text generation.
- Variations.
- Save and reopen projects.
- Template recommendation.
- Library.

## Phase 2 — Visual guidance

- Secure uploads.
- Vision-based feedback.
- Brand-kit extraction from an uploaded logo.
- Template copy placement.
- Export-ready creative brief.

## Phase 3 — Productivity

- Voice input.
- Team comments.
- Reusable campaigns.
- Content calendar.
- Export bundles.
- Provider cost dashboard.

## Phase 4 — Optional intelligence

- Curated trend context.
- RSS or public-source research.
- Location-aware event suggestions.
- Performance feedback from user-entered metrics.

## Phase 5 — Integrations

Only after terms, permissions, and API availability are confirmed:

- Social account connection.
- Scheduling.
- Professional-account insights.
- Publishing approval workflows.

## Prioritization rule

A feature moves forward only when it improves one of these:

- time to first useful artifact,
- output relevance,
- editing and reuse,
- trust and safety,
- presenter-visible value.
