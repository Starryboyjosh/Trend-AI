# MVP demo narrative

## Scenario

The presenter creates a business profile for a fictional coffee shop in Tegucigalpa. The objective is to increase visits from university students through Instagram.

## Demonstration sequence

1. Login screen establishes the HiTrendy identity.
2. Onboarding captures the business profile in under one minute.
3. Home displays quick actions and recent projects.
4. User chooses “Crear publicación”.
5. User writes: “Quiero promocionar una bebida fría nueva para esta semana”.
6. HiTrendy returns a concise Instagram package:
   - hook,
   - caption,
   - CTA,
   - hashtags,
   - visual direction,
   - recommended format.
7. User selects “Más juvenil”.
8. System creates a linked variation without replacing the first.
9. User opens template suggestions and selects a vertical post.
10. User saves the result under “Lanzamiento bebida fría”.
11. User uploads a sample poster and receives structured feedback.

## What the presenter says

“HiTrendy does not replace the owner. It remembers the business context, proposes content, explains what to improve, and organizes the work. The user keeps final control.”

## What must be real

- Onboarding persistence.
- Request-to-response flow.
- Structured result.
- Variation.
- Save project.
- Asset feedback, even if the demo provider uses deterministic rules.

## What may be mocked

- Google OAuth.
- Voice transcription.
- Image generation.
- External trend APIs.
- Social publishing.

Mocked features must be labeled as demo behavior in technical documentation, not presented as completed integrations.
