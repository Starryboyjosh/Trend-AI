# Fair demo script

## Duration

Target: 4–6 minutes.

## Preparation

- Reset demo data.
- Ensure the coffee-shop sample profile exists or complete onboarding live.
- Open the app at login.
- Keep the API health page available in another tab for technical questions.

## Script

### 1. Problem — 20 seconds

“Many small businesses know their product but do not have time or marketing experience to create consistent content.”

### 2. Login and profile — 40 seconds

Show the visual identity and the business setup. Explain that HiTrendy remembers the business, target audience, location, platforms, and tone.

### 3. Main request — 60 seconds

Ask:

> Quiero promocionar una bebida fría nueva para esta semana.

Select Instagram and “Juvenil”.

### 4. Result — 60 seconds

Show the structured result card rather than reading a long chatbot paragraph:

- hook,
- caption,
- CTA,
- hashtags,
- visual suggestion.

Explain that the content is editable and based on the active business profile.

### 5. Variation — 30 seconds

Select “Más corto” or “Más juvenil”. Compare versions and show that user edits are not overwritten.

### 6. Project and template — 45 seconds

Save the artifact as a project. Open template suggestions. Explain that HiTrendy connects the writing assistance with organized creative work.

### 7. Visual review — 45 seconds

Upload a prepared poster. Show strengths and priority improvements.

### 8. Technical close — 30 seconds

“The model is interchangeable. The application validates structured output, stores versions, protects user assets, and also runs in demo mode without external credentials.”

## Avoid during demo

- Claiming guaranteed marketing performance.
- Claiming TikTok/Instagram/X scraping exists.
- Showing unfinished menu items as completed integrations.
- Spending most of the time on the landing page.
- Discussing infrastructure before showing user value.
